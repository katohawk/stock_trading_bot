# 核心交易规则（供优化用）

以下是当前 OKX 涨卖跌买脚本的**核心交易规则**（盈利硬约束、三秒价格确认、市价保护、.monitor_ref.json 持久化），便于其他 AI 或你自己做规则优化时对照修改。**总览文档（给 AI 快速理解整体）**：`docs/AI_OVERVIEW.md`。

---

## 1. 信号规则（何时买、何时卖）

- **参考价**：程序维护「参考价」，持久化在 **`.monitor_ref.json`**（按交易对）。首次运行 = 当前价；**仅在实际成交后**更新（防插针/滑点取消时不更新）。
- **买信号**：当前价 **≤ 参考价 × (1 − ratio/100)** → 发买入信号。
- **卖信号**：当前价 **≥ 参考价 × (1 + ratio/100)** → 发卖出信号；**且**必须通过「盈利硬约束」（见下）才真正下单。
- **观望**：涨跌未达 ratio% 或卖出时利润不足 → 不操作。

---

## 2. 盈利硬约束（卖出强制条件）

- 变量 **avg_cost**：持仓成本，持久化在 `.monitor_ref.json` 的 `avg_cost`。
- **卖出条件**：除「涨了 ≥ ratio%」外，必须满足：  
  **(当前价 − avg_cost) / avg_cost > 0.003**  
  即扣费后仍有至少 **0.3%** 利润，否则不卖出。
- **持仓成本**：买入成交后按「原持仓成本×原数量 + 本次成交价×本次数量」加权更新；卖出清仓后清空。

---

## 3. 三秒价格确认（防插针）

- 信号触发且实盘执行时：**每隔 1 秒取一次价，共 3 次**。
- 若 3 次价格的 **标准差 / 均价 > 0.1%**，判定为插针，**取消本次下单**且不更新参考价。
- 下单使用**3 次取样均价**作为确认后的价格。

---

## 4. 市价保护（滑点）

- **max_slippage**：默认 0.001（0.1%）。下单前获取**买一、卖一**。
- 若 **(卖一 − 买一) / 中间价 > max_slippage**，视为价差过大，**暂缓本次下单**。

---

## 5. 仓位与下单规则

- **买入**：**等额买入**，每次固定金额（默认 50 USDT），由 `--buy-amount-usdt` 指定；不超过可用 USDT×0.98，且 ≥10 USDT 才下单。市价单。
- **卖出**：**全部**卖出该交易对持仓；且必须通过「盈利硬约束」检查。市价单。
- **状态持久化**：每次**成交后**将 `reference_price`、`avg_cost`、`position_qty` 写入 **`.monitor_ref.json`**。

---

## 6. 执行与安全

- **API**：获取价格/下单失败时自动重试（如遇 429/限频则指数退避重试）。
- **API Key**：从 `.env` 读取 `OKX_API_KEY`、`OKX_API_SECRET`、`OKX_PASSPHRASE`。
- **日志**：详细中文日志（前缀 `[日志]`），含触发原因、盈利/插针/滑点检查结果、成交与预期收益率/扣费后收益。

---

## 7. 可调参数汇总

| 参数 | 含义 | 默认 |
|------|------|------|
| --symbol | 交易对 | BTC/USDT |
| --ratio | 触发比例（%） | 0.5 |
| --interval | 轮询间隔（秒），0 表示只跑一次 | 0 |
| --taker-fee-rate | 吃单费率 | 0.001 |
| --buy-amount-usdt | 每次买入固定 USDT 金额 | 50 |
| --max-slippage | 市价保护：买一卖一价差上限 | 0.001 |
| reference_price / avg_cost / position_qty | 参考价、持仓成本、持仓量 | 持久化于 .monitor_ref.json |

脚本内常量（改规则时可直接改）：`MIN_PROFIT_RATIO = 0.003`、`PRICE_SAMPLES = 3`、`PRICE_SAMPLE_INTERVAL = 1.0`、`SPIKE_STD_THRESHOLD_PCT = 0.001`、`MAX_SLIPPAGE = 0.001`。
