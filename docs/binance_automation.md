# 钱在币安怎么办：自动化方案与备选平台

资金在**币安（Binance）**时，可以直接用 API 做自动化；若你所在地区无法用币安或 API 不稳定，可转到下面列出的替代平台。

---

## 一、币安上直接自动化（推荐优先试）

### 1. 可行性

- 币安提供**现货 API**（REST + WebSocket），本仓库用 **ccxt** 统一调用，已实现：
  - 拉 K 线（`CryptoAdapter`）
  - 现货下单、查余额、查订单（`BinanceBroker`）
- **资金不用转出**，在币安现货账户即可：API 只做“查行情 + 下单”，资金仍在你的币安账户。

### 2. 步骤概要

1. **创建 API Key**
   - 币安 App 或网页：个人中心 → API 管理 → 创建 API。
   - 权限只勾选：**启用现货与杠杆交易**（不要开提现，降低风险）。
   - 保存 **API Key** 和 **Secret**，Secret 只显示一次。

2. **（可选）IP 白名单**
   - 若开启，只填你跑脚本的服务器或电脑公网 IP，避免 Key 泄露后被异地调用。

3. **本仓库用法**
   - 安装：`pip install ccxt`
   - 环境变量（推荐）：`export BINANCE_API_KEY=xxx`、`export BINANCE_API_SECRET=xxx`
   - 代码里用 `BinanceBroker(api_key=os.environ["BINANCE_API_KEY"], api_secret=os.environ["BINANCE_API_SECRET"])` 接在执行层，用同一套“策略 + 监控”逻辑，只是把下单交给币安。

4. **测试网**
   - 币安有现货测试网，可先用测试环境验证再上实盘：`BinanceBroker(..., testnet=True)`，并在币安测试网页面拿测试用 API Key。

### 3. 限制与注意

- **地区**：部分国家/地区无法访问币安或只能用 Binance.US 等分站，若你那边打不开官网或 App，就考虑下面备选平台。
- **频率**：API 有请求权重限制，高频下单可能被限流；本仓库的定时监控（如每 5 分钟一次）一般没问题。
- **安全**：API Key 不要写进代码、不要提交到 Git；用环境变量或本地配置文件并加入 `.gitignore`。

---

## 二、若币安不行：转到哪些平台更方便自动化

若你**无法使用币安**（地区、KYC、或 API 经常异常），可把资金转到下面任一平台，再用同样方式（ccxt + 自写或本仓库逻辑）做自动化。

| 平台 | 说明 | 自动化友好度 |
|------|------|--------------|
| **OKX（欧易）** | 支持多地区，API 文档全，ccxt 支持好 | 高，和币安类似 |
| **Bybit** | 现货+合约都有，API 清晰 | 高 |
| **Gate.io** | 老牌交易所，ccxt 支持 | 中高 |
| **Kraken** | 合规较好，欧美常用 | 中高 |
| **Coinbase** | 合规交易所，有 API，ccxt 支持 | 中（更多面向美股/合规） |
| **Bitget** | 合约/现货都有，API 可用 | 中高 |

**建议**：  
- 若你在**国内**：合规上不能宣传具体所，只能自行了解当前可访问的交易所；从纯自动化角度，**OKX、Bybit、Gate.io** 等用 ccxt 都能类似币安一样接进本仓库。  
- 若你在**海外**：可优先试币安 → 不行再 OKX / Bybit / Kraken。

---

## 三、不想写代码：用“现成工具”接币安

资金在币安，但不想自己写/维护代码，可以用**第三方自动化工具**连币安 API（你把 API Key 配在工具里，资金仍在币安）：

| 类型 | 例子 | 说明 |
|------|------|------|
| 网格/定投机器人 | **3Commas**、**Cryptohopper**、**Pionex** | 在平台里选“连接 Binance”，用他们的网格/策略，币安账户授权 API 即可 |
| 跟单/策略托管 | 部分交易所自带或第三方 | 策略在别人服务器跑，你只连 API |

注意：  
- 这类工具会**代你下单**，API 权限要给“交易”，务必选可信产品、并设好 IP/权限限制。  
- 若你**不想把 API 给第三方**，就只能本机或自己的服务器跑本仓库这类自建逻辑（钱仍在币安，API 只在你自己的环境用）。

---

## 四、本仓库里的对应关系

- **数据（K 线 / 当前价）**：`CryptoAdapter(exchange_id="binance", ...)`，需 API 时可传 key/secret，否则只读行情部分接口可能有限制。  
- **执行（下单）**：`BinanceBroker(api_key=..., api_secret=...)`，实现 `BrokerBase`，可替换 `run_live.py` 里的 SimulatedBroker，实现“涨卖跌买”或网格的自动下单。  
- **监控只推荐不下单**：继续用 `run_monitor.py`，不配币安 API 也行，用 yfinance 等拿到价格即可。

总结：**钱在币安 → 优先用币安 API + 本仓库 BinanceBroker 自动化**；若币安不可用，再考虑转到 OKX/Bybit 等并用同一套思路对接；若不想写代码，再用 3Commas 等工具连币安。
